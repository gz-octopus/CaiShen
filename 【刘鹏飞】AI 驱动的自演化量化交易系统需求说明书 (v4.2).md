
## 1. 项目愿景 (Vision)

构建一个“**懂金融、能进化、易交互**”的智能量化研究与交易执行闭环平台。系统旨在通过自然语言交互大幅降低量化策略的开发门槛，使研究员能够通过语义指令快速验证投资灵感。核心目标是建立一套具备“研究记忆”的自动化体系，能够在动态波动的市场中，通过不断的自我实验与逻辑修正，自主发现并演化出具备强鲁棒性的交易策略，最终实现从投资直觉到严谨数理执行的无缝闭环。

## 2. 核心设计原则

- **混合架构 (Hybrid)**：自研灵活的任务编排层，深度集成高性能存储、极速向量化回测以及最前沿的大模型推理组件。
- **研交解耦**：因子研究系统（离线）负责挖掘因子表达式；交易执行系统（在线）负责高性能算子解析与实时下单。
- **5s 亚分钟级驱动**：系统必须支持 5 秒量级的实时行情对齐、因子计算及 AI 推理。
- **表达式驱动**：所有策略逻辑以标准的 Qlib 算子表达式形式进行传递和存储。
- **闭环自演化 (Self-Evolving)**：超越单纯的参数寻优，通过 **“检索-生成-评价-蒸馏”** 的全流程循环，实现策略逻辑与因子结构的自我迭代。
- **抗过拟合 (Anti-Overfitting)**：通过生成多维度的模拟市场数据进行压力测试，并结合“经验记忆库”排除逻辑不可解释的因子。
- **务实开发 (Developer-Centric)**：初期优先适配 **CPU 多线程加速**，通过多种低门槛渠道获取全球市场行情，无GUI生产，确保系统在 Windows 环境下即可实现快速原型运转。

## 3. 功能模块详细需求

### 3.1 数据中枢 (Data Hub)

- **多级行情订阅：
  
    - **富途 API (Futu OpenD)**：原生支持接入富途行情接口，同步外盘期货及国内市场的实时与历史数据。
    - **MT5 接口 (MT5 SDK)**：利用模拟账号一键同步 HSI（恒指）、GC（美黄金）等标的的 1min 级长周期历史数据。
    - **AkShare 备份**：作为国内数据源的补充，确保数据获取的零死角。
    - **内存滑动窗口**： 维护内存滑动窗口（Sliding Window），实现 5 秒频率的数据对齐。

- **数据缓冲机制**：
	- 建立行情缓冲区，确保计算逻辑不阻塞 API 接收数据线程。
- **二进制高效存储**：将多源数据统一转化为高性能二进制格式，利用内存映射（mmap）技术支撑百万级以上数据量下的极速读取。
  

### 3.2 NLP 策略研究与交互 (The Interface)

- **逻辑翻译引擎 (Gemini 3.1 Pro 驱动)**：
  
    - 利用最新的 Gemini 3.1 Pro API 将自然语言指令（如：“寻找波动率与成交量背离的信号”）准确转化为标准化的**算子表达式**或 Pandas 计算代码。
    
- **开发者友好体验**：所有研究动作通过 Jupyter Notebook 代码块触发，实现“对话即策略，代码即绘图”。
  

### 3.3 核心智能体与进化引擎 (The Brain & Muscle)

采用 **“QuantaAlpha 的灵魂 + FactorMiner 的肌肉”** 的混合架构：
- **QuantaAlpha：
	- **以 QuantaAlpha 为中枢**：利用其进化逻辑（Mutation & Crossover）作为 Agent 的核心决策机制，通过变异和交叉产生新因子，自动在历史数据及 Kronos 合成数据上通过 LLM 进行因子算子组合变异 。
	- -产出符合 Qlib 语法规范的因子定义字符串（Factor Expressions）。
- **FactorMiner：
	- **挂载 FactorMiner 算子库**：在 QuantaAlpha的`operators.py` 中引入 FactorMiner 那套 GPU 加速的 C-compiled 算子，以支撑 HSI M1 级别的高频计算需求 。
	- **经验沉淀**：参考 FactorMiner 的 SQLite 记忆库设计，记录因子的逻辑表达式、多维度绩效（夏普、胜率、回撤）及失败归因总结，防止 Agent 在后续代际中重复无效的变异。

### 3.4 回测系统

- **历史数据生成 (Kronos)：
    - 根据历史数据，生成100倍类似的数据，减少回测产生的过拟合。
- **极速回测 (VectorBT)**：
    - 利用向量化计算，在秒级完成对演化变体的性能评估，支撑进化算法的高频迭代需求。

### 3.5 实盘决策引擎 (Online Alpha Runner)
考虑引入[vn.py](https://github.com/vnpy/vnpy)的决策引擎，并加入以下特性：

- **动态因子解析**：
  
    - 系统启动时从因子库读取表达式字符串。
      
    - 在内存中将表达式映射为向量化算子（NumPy/Polars 加速），无需重新编译。
    
- **5秒决策流水线**：
  
    - 每 5 秒触发一次“数据采样 -> 因子计算 -> 信号生成”。
    
- **Kronos 智能拦劫（The Filter）**：
  
    - 当因子产生买卖信号时，强制唤醒 Kronos 轻量化模型。
      
    - 根据实时形态输出预测概率分布，若概率低于设定的准入阈值，则拦截该信号。
      

### 3.6 交易执行与风险控制 (Execution & RMS)
考虑引入[vn.py](https://github.com/vnpy/vnpy)
- **统一网关管理器**：
	- **适配器模式**：抽象出适配层，支持多家交易商的api。
	- **多通道并发**：系统启动时，根据配置文件同时初始化多个 API 连接。每个网关在独立的线程/异步任务中维护心跳。
	- **指令转换 (Protocol Translation)**：网关层负责将标准信号转换为对应 API 的指令
- **订单状态机管理**：
    - 维护 Order Manager，实时跟踪下单、撤单、补单及成交状态。
      
    - 处理 5 秒频率下的并发订单冲突逻辑。
    
- **实盘风控 (Pre-Trade RMS)**：
  
    - **静态检查**：资金占用上限、品种白名单、交易时间段限制。
      
    - **动态检查**：单笔最大滑点保护、5秒内下单频率限制。
    
- **持仓同步逻辑**：
    - 自动与富途等交易商后台同步持仓，解决“程序记账”与“柜台记录”的不一致问题。

## 5. 执行路线图 (Roadmap)

### 第一阶段：构建“本地数据湖”与 Qlib 转换链路 (基础架构期)

**核心目标**：解决 5s 级高频数据的存储效率与读取吞吐问题。

1. **Parquet 存储持久化**：开发 `market_recorder.py`，通过 Futu/MT5 订阅 HSI/MHI 实时行情，每 5 分钟将内存数据刷入 **Parquet** 文件。
   
2. **Qlib 二进制转换算盘**：实现 `parquet_to_qlib.py` 工具，定期（如每日收盘后）将 Parquet 增量数据转化为 Qlib 的高性能 **.bin** 格式。
   
3. **多线程计算验收**：在 Jupyter 中调用 Qlib Engine，验证对 100 万行 5s 数据进行 `Mean(Close, 20) / Close - 1` 这类因子计算的耗时，目标应在 **100ms** 以内。
   
4. **验收标准**：本地 `data/` 目录下形成以 `.parquet` 结尾的历史归档和以 `.bin` 结尾的 Qlib 训练数据。
   

### 第二阶段：NLP 驱动的因子研究闭环 (界面与交互期)

**核心目标**：实现“对话即策略”，打通从自然语言到因子回测的路径。

1. **算子 Prompt 框架**：针对 Gemini 3.1 Pro 构建专门的 Prompt，包含 Qlib 算子手册（如 `Ref`, `Std`, `Slope` 等）和 FactorMiner 的 C-compiled 算子规范。
   
2. **指令翻译引擎**：自然语言指令 $\rightarrow$ Gemini 3.1 Pro $\rightarrow$ Qlib Expression。
   
3. **VectorBT 极速评估**：接收 Qlib 计算出的 Factor 值，通过 VectorBT 进行向量化回测，输出 $SharpeRatio$、最大回撤等核心指标。
   
4. **验收标准**：在 Jupyter 输入“帮我测试恒指 5s 级别下，成交量极值反转的因子”，系统自动绘制回测曲线并给出算子表达式。

### 第三阶段：FactorMiner 进化记忆与自演化逻辑 (智能体核心)

**核心目标**：让 AI 具备“研究记忆”，实现因子的自主迭代。

1. **SQLite 经验记忆库**：建立 `experience.db`，记录每个因子的：
   
    - 表达式逻辑（Logic String）
      
    - 回测绩效数据（Metrics）
      
    - 失败原因（Failure Attribution，由 Gemini 生成）。
    
2. **QuantaAlpha 进化算法**：启动变异（Mutation）与交叉（Crossover）任务，AI 在尝试新因子前，先检索 SQLite 库，排除与已知失败逻辑相似的变体。
   
3. **Kronos 数据合成**：利用 Kronos 根据 HSI 历史特征生成 100 倍模拟数据，对挖掘出的高夏普因子进行“压力测试”，剔除过拟合因子。
   
4. **验收标准**：系统能连续运行 24 小时，自主生成并记录超过 500 个候选因子，并筛选出前 5% 进入备选池。
   

### 第四阶段：5s 实盘决策与 Kronos 拦劫 (执行与防御期)

**核心目标**：实现从“离线研究”到“在线交易”的低延迟衔接。

1. **vn.py 策略网关适配**：在 vn.py 架构下实现 `OnlineAlphaRunner`，系统启动时动态加载第三阶段产出的 Top 因子表达式。
   
2. **实时 5s 流水线**：
   
    - `T+0s`: 收到行情推送。
      
    - `T+1s`: 调用内存算子计算当前因子值。
      
    - `T+2s`: 生成信号，并触发 **Kronos 概率过滤**。
      
    - `T+3s`: 若概率通过阈值，下达订单。
    
3. **实盘风控监控**：在 Windows 端利用可视化工具监控持仓、成交及 5s 延迟波动。
   
4. **验收标准**：模拟盘/小单实盘连续运行 1 周，信号生成频率与执行逻辑符合预期。
